This chart does not run landlock-genprof itself — it's a CLI/kubectl-plugin
tool, invoked on demand from outside the cluster (or as a kubectl plugin),
not a long-running workload. Prerequisite: Inspektor Gadget must already be
deployed in the "{{ .Values.gadget.namespace }}" namespace (kubectl gadget deploy).

Try it:
  kubectl auth can-i get pods --as=system:serviceaccount:{{ .Values.namespace.name }}:{{ .Values.serviceAccount.name }}

Next: landlock-genprof trace --pod <pod> --binary <path> --duration 60s
See README.md and docs/roadmap.md for the full workflow.

Upgrading later? This chart's CRDs live under crds/ — Helm installs them on
first install only and never updates them on `helm upgrade` (by design, see
https://helm.sh/docs/chart_best_practices/custom_resource_definitions/).
If a release changes the CRD schema, re-apply it yourself:
  kubectl apply -f {{ .Chart.Name }}/crds/
